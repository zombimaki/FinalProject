Using Machine Learning Models To Predict Crime Incidents



Our goal was to use machine learning (ML) to evaluate Minneapolis crime data from 2010 through 2020 and then use any effective models to make predictions about the number of crime incidents in 2021 and beyond.



Data preparation and cleaning

The data was obtained from the City of Minneapolis Open Data Portal where crime incident data for each year is available in csv format. This data source includes all property crimes and violent crimes, using the FBI uniform crime reporting codes to categorize each incident. (More info about UCR codes) Note that this data does not include what are categorized as type II offenses: traffic stops, disorderly conduct, DWI, drug offenses, etc. That data is tracked seperately from the type I incidents that we are evaluating in our models.


We noted that the Minneapolis Police Deparment (MPD) changed their software in the middle of 2018 which impacted the structure of the csv files (column names changed; the case number format changed, and dates and times were formatted differently). We used a python script to combine the CSVs and then used pandas to unify the formatting and data types of each column. The final combined data set was exported as a CSV file for use in our models.
